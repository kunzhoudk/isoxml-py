# ISOXML 应用图类型比较

ISOXML支持三种不同类型的应用图（Application Map），用于精准农业中的变量施用作业。本文档对比这三种类型的特点和适用场景。

## 概览对比

| 特性 | 矢量型（Vector） | 网格型 Type 1 | 网格型 Type 2 |
|------|-----------------|--------------|--------------|
| **数据结构** | 多边形（Polygon） | 规则网格 + 查找表 | 规则网格 + 直接编码 |
| **数据存储** | XML中存储几何和值 | XML中存储所有值 | 二进制文件存储值 |
| **适用场景** | 少数处理区域 | 少数不同值 | 大量连续值 |
| **地理表示** | 精确的多边形边界 | 栅格单元格 | 栅格单元格 |
| **XML文件大小** | 中等 | 可能很大 | 较小（仅含缩放信息） |
| **处理方式** | 按区域整体施用 | 按单元格映射值 | 按单元格直接读值 |
| **值存储原理** | 直接存储 | 查找表（Lookup table） | int32编码 + scale/offset |

## 详细说明

### 1. 矢量型（Vector）

**示例文件**: `app_map_vector.py`

**工作原理**:
- 每个处理区域（TreatmentZone）包含：
  - 多边形边界（geometry）
  - 施用参数（dose/DDI value）
  - 唯一代码（code）
- 所有数据存储在XML的TreatmentZone元素中

**优势**:
- 精确的区域边界定义
- 结构清晰，易于理解
- 适合简单的分区管理

**劣势**:
- 不适合处理大量细分区域
- 复杂边界会导致XML文件较大

**典型应用场景**:
- 变量施肥（少数区域）
- 点喷应用（Spot sprayer）
- 基于土壤类型的分区作业
- 有机农业的分区管理

**硬件测试**:
- ✅ New Holland T7 + IntelliView 12 + Kverneland iXter B18
  - 注意：终端无法选择DDI

### 2. 网格型 Type 1

**示例文件**: `app_map_grid_type_1.py`

**工作原理**:
- 使用规则网格划分地块
- 每个网格单元格存储一个代码（code）
- 代码通过查找表映射到实际值
- 实际值存储在XML的TreatmentZone的ProcessDataVariable中

**数据流程**:
```
网格单元格 → code值(0,1,2,3...) → TreatmentZone → ProcessDataVariable → 实际施用量
```

**优势**:
- 适合离散的值（如几个固定的施用等级）
- 网格结构规整，易于处理

**劣势**:
- 如果不同值太多，XML文件会变得很大
- 查找表方式增加了一层间接性

**典型应用场景**:
- 固定等级的施肥策略（低/中/高）
- 简单的分级灌溉
- 土壤改良剂的分级施用

**数据示例**:
```python
grid_data = np.array([[0, 1, 2, 3]])
# code 0 → 1 kg/ha
# code 1 → 1000 kg/ha
# code 2 → 2000 kg/ha
# code 3 → 3000 kg/ha
```

### 3. 网格型 Type 2

**示例文件**: `app_map_grid_type_2.py`

**工作原理**:
- 使用规则网格划分地块
- 实际值以int32格式直接写入二进制文件
- XML只包含缩放（scale）和偏移（offset）信息
- 读取时：`实际值 = (二进制值 * scale) + offset`

**数据流程**:
```
网格单元格 → 二进制文件中的int32值 → 应用scale/offset → 实际施用量
```

**优势**:
- 适合大量连续变化的值
- XML文件保持较小
- 存储效率高

**劣势**:
- 需要生成额外的二进制文件
- 相对复杂的数据处理

**典型应用场景**:
- 精准变量施肥（基于土壤养分图）
- 连续变化的灌溉量
- 基于遥感数据的变量播种
- 高分辨率处方图

**数据示例**:
```python
grid_data = np.array([
    [0, 1111],
    [2222, 3333]
])
# 每个值直接对应施用量（考虑DDI的分辨率）
```

**硬件测试**:
- ✅ Deutz-Fahr 6140.4 + Bogballe L20W
- ⚠️ New Holland T7 + IntelliView 12 + Kverneland iXter B18
  - 注意：TC（Task Controller）不接受设定值

## 选择建议

### 使用矢量型（Vector），如果：
- ✓ 处理区域数量少（<20个）
- ✓ 每个区域有明确的边界
- ✓ 区域之间差异明显
- ✓ 需要简单直观的数据结构

### 使用网格型 Type 1，如果：
- ✓ 只有少数几个不同的施用等级（<10种）
- ✓ 需要规则的网格结构
- ✓ 值是离散的而非连续的
- ✓ XML文件大小可接受

### 使用网格型 Type 2，如果：
- ✓ 有大量不同的数值
- ✓ 值连续变化（如基于遥感数据）
- ✓ 需要高分辨率处方图
- ✓ 希望XML文件保持较小

## 重要注意事项

### 坐标系统
- ISOXML网格坐标系统是**从下到上**（bottom-up）
- 在终端/查看器上显示时，图像可能会上下颠倒

### DDI（Data Dictionary Item）
- 所有类型都需要指定DDI来定义数据类型（如肥料类型、施用量单位等）
- DDI ID 6 通常用于演示示例

### 硬件兼容性
- 不同的农机终端对三种类型的支持程度可能不同
- 建议在实际应用前在目标设备上测试
